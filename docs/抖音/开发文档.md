## 一、今日开发内容总结（2026-04-17）

### 1. 人群预设（Audience Presets）

- **配置**：`config/audience_presets.yaml`，服务端 merge / `prepare_task_params` / `strict_task_params`。
- **API**：`GET /task-params/audience-presets`（etag / if_etag）、`POST /task-params/reload-audience-presets`。
- **前端**：`_ocReloadAndSeedAudiencePresets` 统一 POST 重载 + platforms 缓存 + flow 客户端种子（`overview.js` / `platforms.js` / `tiktok-ops.js`）。

### 2. 关键 Bug 修复（overview ↔ tiktok-ops）

- **重复顶层声明**（`loadTtOpsPanel`、`let _ttCtaUrl`、`ttSetCtaUrl`、`_ttShowPitchModal` 等）导致 **`tiktok-ops.js` 整段解析失败**。
- **处理**：删除 overview 中重复的 `loadTtOpsPanel` 壳；tiktok-ops 合并壳层逻辑；去除重复 `_ttCtaUrl`/`ttSetCtaUrl`；批量话术弹窗改名为 **`_ttShowBatchPitchModal`**。

### 3. E2E（Playwright）

- **`requirements-e2e.txt`**、`tests/e2e/conftest.py`（子进程 uvicorn、**`e2e_guest_token` 会话级**）。
- **单会话三条入口**：运维条 → 平台弹窗 → 总览「执行方案」「↻ 人群预设」，各 **1× POST** `reload-audience-presets`（避免二次 `page.goto` 与 uvicorn 竞态）。
- **`pytest.ini`**：默认 **`-m "not e2e"`**，全量单元测试无需 Playwright。

### 4. CI

- **E2E job**：Playwright 缓存、`timeout-minutes: 30`、`--tracing=retain-on-failure`、失败上传 **`test-results/`**。
- **`.gitignore`**：`test-results/`、`playwright-report/`。

### 5. 其他文档

- **`docs/抖音/操作日志.md`**：操作与变更摘要。
