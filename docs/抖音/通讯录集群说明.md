# 通讯录：主控 / Worker 与标准视图说明

<!-- 面向运维、市场与一线：与前端「标准通讯录视图」横幅配套 -->

## 一、现象（给市场 / 用户）

- **标准通讯录视图**：名单、搜索、导入/导出仍可用；「匹配 APP / 已打招呼」等**智能统计**可能显示为「—」，表示**增强数据未加载**，不等于业务量为零。
- **加载未成功**（红色）：无法拉取基础列表，需联系管理员检查节点与网络。

## 二、技术排障（给开发 / 运维）

### 工单信息模板

1. 设备 `device_id`（完整序列号）
2. 主控与目标 Worker 的 **Git/版本**是否一致
3. 浏览器 Network：`GET .../contacts/enriched` 状态码（200 / 404 / 502）
4. 主控日志是否含：`Worker 对增强通讯录返回 404`（见 `worker_device_proxy`）

### 常见原因

| 情况 | 处理 |
|------|------|
| Worker 未部署 `contacts/enriched` | 对齐代码并重启 Worker |
| 转发 gzip 乱码（历史问题） | 已用主控侧解压与 `Accept-Encoding: identity` 修复 |
| 设备不在任何节点 | 检查 `cluster_state` 心跳、`device_aliases` 的 `host_name` |

## 三、集群探针 API + 仪表盘入口（已实现）

- **GET** `/system/cluster/probe-contacts-enriched`（需 `X-API-Key` 或与登录会话一致）
- 返回 `local` / `workers[]`：各节点 `openapi.json` 是否含路径片段 `contacts/enriched`，以及抽样 path 列表。
- `summary.any_node_has_enriched`：任一节点的 OpenAPI 声明了增强通讯录即 `true`（不代表某台手机一定可用，仅版本线索）。
- **仪表盘**：TikTok 总览命令条增加 **「探针」** 按钮，弹窗展示上表（可刷新 / 重试）。

## 四、全局 API 错误提示（core.js）

- HTTP 错误体若为 JSON 且 `detail` 为对象，优先展示 **`message`**、**`hint`**、**`code`**（便于识别 `device_not_on_host` 等）。

## 五、下一阶段可改进（产品 / 工程）

- Worker 升级后与主控再跑一次探针，确认各 Worker **enriched** 为 ✓。
- 市场物料：单独一页「标准视图 vs 智能视图」截图对照。
