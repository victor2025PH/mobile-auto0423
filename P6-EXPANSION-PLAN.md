### 2.2 关键创新：自学习选择器系统

```
第一次操作新 App:
  1. 截图 → OmniParser V2 检测所有 UI 元素
  2. LLM 理解: "这是消息发送按钮，坐标 (650, 1350)"
  3. 同时 dump XML，匹配坐标附近的 resource-id/content-desc
  4. 如果匹配到选择器 → 自动保存到 selectors.yaml
  5. 下次直接用选择器（快速、免费）

第 N 次操作:
  选择器命中 → 直接操作（零 Vision 消耗）
  选择器失败 → Vision 重新识别 → 更新选择器
```

这意味着：**新 App 上线只需安装到手机，系统自动学习如何操作它**。

## 四、技术选型决策

### 4.1 为什么选 OmniParser V2 而不是纯 u2 选择器？

| 维度 | 纯 u2 选择器 | OmniParser V2 + u2 混合 |
|------|-------------|------------------------|
| 新 App 接入速度 | 2-3 天（需手动 dump + 编写选择器） | **15 分钟**（安装 App → 自动学习） |
| UI 更新适应 | 版本更新后选择器失效，需人工修复 | **自动适应**（Vision 重新识别） |
| 操作成本 | 零（纯设备端） | 每次 Vision 调用约 $0.005 |
| 操作速度 | 快（< 1s） | 首次慢（3-5s），学习后同样快 |
| 适用范围 | 需要 resource-id 的 App | **任何 App，包括游戏、银行 App** |

**结论**：混合模式。核心 App 用选择器（快），新 App 或选择器失败时用 Vision（智能），Vision 成功后自动学习选择器（进化）。

### 4.2 设备规划

| 规模 | 设备数 | App 覆盖 | 日获客量 |
|------|--------|---------|---------|
| 当前 | 2 台 | 3 App | 50-100 |
| Phase 6 | 4-6 台 | 6-8 App | 300-500 |
| 未来 | 10+ 台 | 10+ App | 1000+ |

每台设备可同时安装所有 App，通过调度器轮换执行不同平台的任务。

### 4.3 合规红线（新平台）

| 平台 | 每日操作上限 | 检测重点 | 对策 |
|------|------------|---------|------|
| **Facebook** | 30 好友请求/天，50 消息/天 | IP 行为、设备指纹、操作频率 | 手机原生操作 + 自然间隔 |
| **Instagram** | 60 关注/天，100 赞/天，20 DM/天 | 频率突变、机器行为模式 | GramAddict 已验证的行为参数 |
| **TikTok** | 30 关注/天，50 评论/天 | 滑动速度、观看时长 | 视频实际观看 3-8s 后操作 |
| **X/Twitter** | 50 关注/天，100 赞/天，40 DM/天 | API 与 UI 混合检测 | 纯 u2 UI 操作 |

## 五、预期收益

### 5.1 获客效率对比

| 方式 | 每日触达 | 回复率 | 转化率 | 每日新客 |
|------|---------|--------|--------|---------|
| 手动 | 30-50 | 15% | 3% | 0.5-1.5 |
| 当前系统（3 App） | 100-200 | 15% | 3% | 3-6 |
| **P6 扩展（8 App）** | **500-800** | **18%**★ | **4%**★ | **20-32** |

★ 多平台触达同一客户提升回复率，AI 个性化消息提升转化率

### 5.2 核心竞争力

1. **自学习 UI 引擎** — 任何新 App 15 分钟可对接
2. **多平台协同获客** — 同一客户在 LinkedIn 触达 → Instagram 互动 → WhatsApp 深聊
3. **AI 驱动对话** — 每条消息独特，自动意图分析，自动升级沟通
4. **合规第一** — 每个平台独立配额，人类行为模拟，零封号风险
